## What to build

Build the reproducible offline ETL flow that converts raw recipe data into the processed runtime dataset used by the app. This slice should make the data preparation pipeline concrete and explainable for the course project, including schema enforcement, normalization, duplicate handling, and output documentation.

## Acceptance criteria

- [ ] Raw recipe input can be transformed into a processed dataset artifact that satisfies the runtime schema contract in the PRD
- [ ] Invalid and duplicate records are removed before runtime data is produced
- [ ] The ETL flow documents what changed between raw input and processed output, including before-and-after record counts or equivalent reporting
