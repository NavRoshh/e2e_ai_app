## What to build

Make recommendation explanations inspectable and trustworthy by rendering deterministic explanation data on each recipe card. This slice should ensure users can see score, matched ingredients, missing ingredients, and a template-based reason for every recommendation returned by the app.

## Acceptance criteria

- [ ] Each recommendation includes deterministic explanation fields suitable for display in the UI
- [ ] The UI renders those explanation fields clearly for each returned recipe
- [ ] Per-recipe explanations are template-based and do not depend on the LLM layer
